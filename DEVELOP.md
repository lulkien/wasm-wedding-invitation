# Developer Guide

## Prerequisites

| Tool | Version | Install |
|---|---|---|
| Rust | stable | `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs \| sh` |
| Dioxus CLI | 0.7.x | `cargo install dioxus-cli` |
| wasm32 target | — | `rustup target add wasm32-unknown-unknown` |

---

## Getting Started

```sh
git clone https://github.com/lulkien/wasm-wedding-invitation
cd wasm-wedding-invitation
```

The build script compiles SCSS automatically on every `cargo build` / `dx serve`.
No manual SCSS compilation step is needed.

### Run in development

```sh
dx serve
```

Starts a fullstack dev server (client on WASM + server on native).
Hot-reload is enabled. Open `http://localhost:8080`.

### Build for production

```sh
dx build --release
```

Output lands in `dist/`.

## Styling

All styles live in `assets/scss/`. `assets/main.css` is generated by `build.rs`
and is gitignored — never edit it directly.

### Design tokens

All colours, fonts and layout constants are in `_variables.scss`:

```scss
$color-primary:   #996e6d;   // dusty rose — buttons, links, headings
$color-bg-pink:   #f4e2e2;   // time + RSVP section background
$color-bg-pink-dark: #edd2d1; // location section background

$font-body:    'Raleway', Tahoma, sans-serif;
$font-display: 'Rouge Script', sans-serif;  // couple names, hero title

$navbar-height: 50px;
$section-pad-v: 80px;
$section-pad-h: 40px;
```

Changing a variable here propagates to every partial automatically.

### Trigger a SCSS recompile

Any `cargo build` or `dx serve` rerun will recompile SCSS if any file under
`assets/scss/` has changed (tracked via `cargo:rerun-if-changed=assets/scss/`).

## Database

The app uses **SurrealDB** as an external server process, connected over WebSocket
from the Dioxus server binary. See [SURREALDB.md](SURREALDB.md) for the full
native installation and deployment guide.

### Connection constants

All connection details live in `src/database/mod.rs` under `#[cfg(feature = "server")]`:

| Constant | Default | Description |
|---|---|---|
| `DB_URL` | `ws://127.0.0.1:8000` | WebSocket URL of the SurrealDB instance |
| `DB_NS` | `wedding` | Namespace |
| `DB_NAME` | `wedding` | Database name |
| `DB_USER` | `root` | Username |
| `DB_PASS` | `secret` | Password |

The connection is initialised lazily on first use via `tokio::sync::OnceCell` — no
explicit startup step required.

### Start SurrealDB (development)

```sh
surreal start --user root --password secret --bind 127.0.0.1:8000 memory
```

### Schema

Run once against a fresh instance:

```sql
-- Departure location lookup
DEFINE TABLE location_map SCHEMAFULL;
DEFINE FIELD location_id   ON location_map TYPE int;
DEFINE FIELD location_name ON location_map TYPE string;

CREATE location_map:1 SET location_id = 1, location_name = 'FPT Tower';
CREATE location_map:2 SET location_id = 2, location_name = 'Handico Tower';
CREATE location_map:3 SET location_id = 3, location_name = 'Lotte Mall West Lake';
CREATE location_map:4 SET location_id = 4, location_name = 'I use my own vehicle';
CREATE location_map:5 SET location_id = 5, location_name = 'I\'ll pass';

-- Guest table
DEFINE TABLE people SCHEMAFULL;
DEFINE FIELD uid          ON people TYPE string ASSERT $value != NONE;
DEFINE FIELD name         ON people TYPE string;
DEFINE FIELD greeting     ON people TYPE string;
DEFINE FIELD line1        ON people TYPE string;
DEFINE FIELD line2        ON people TYPE option<string>;
DEFINE FIELD line3        ON people TYPE option<string>;
DEFINE FIELD desc         ON people TYPE option<string>;
DEFINE FIELD depart_from  ON people TYPE option<record<location_map>>;
```

### `depart_from` record links

`depart_from` is a SurrealDB record link to `location_map`, not a plain integer.
`NONE` means the guest has not yet chosen a departure point.

| Record | `location_id` | Meaning |
|---|---|---|
| `NONE` | — | Not decided yet (default) |
| `location_map:1` | `1` | FPT Tower shuttle |
| `location_map:2` | `2` | Handico Tower shuttle |
| `location_map:3` | `3` | Lotte Mall West Lake shuttle |
| `location_map:4` | `4` | Own vehicle |
| `location_map:5` | `5` | Can't attend |

### Add a guest

Each guest gets a unique 8-character hex UID. Generate one with:

```sh
openssl rand -hex 4
# e.g. a1b2c3d4
```

Then insert via the SurrealDB CLI (record key = uid):

```sql
CREATE people:a1b2c3d4 CONTENT {
    uid:         "a1b2c3d4",
    name:        "Nguyen Van A",
    greeting:    "Dear",
    line1:       "We joyfully invite you to share in the celebration of our wedding.",
    line2:       "Your presence would mean the world to us.",
    line3:       NONE,
    desc:        "optional internal note",
    depart_from: NONE
};
```

The guest's unique invitation URL is:

```
https://your-domain.com/invitation/a1b2c3d4
```

## Configuration

Edit `config/wedding.json` — no code changes needed for ceremony details:

```json
{
  "ceremony": {
    "date_utc": "2026-03-28T06:30:00Z",
    "date_display": "Saturday, March 28, 2026",
    "day_of_week": "Saturday",
    "day_number": "28",
    "month_year": "March 2026",
    "reception_time": "8:30 AM",
    "ceremony_time": "1:30 PM"
  },
  "venue": {
    "name": "Near Doc Suoi Roundabout",
    "address": "...",
    "location_line": "...",
    "province": "Hung Yen Province",
    "maps_embed_url": "https://www.google.com/maps/embed?...",
    "maps_directions_url": "https://maps.app.goo.gl/..."
  },
  "couple": {
    "groom_name": "Luu Hoang Kien",
    "bride_name": "Pham Thi Hang",
    "title": "Hoang Kien & Pham Hang"
  }
}
```

> `date_utc` drives the live countdown. Use ISO 8601 UTC — Vietnam is UTC+7, so
> 1:30 PM ICT = `06:30:00Z`.

A rebuild is required after changing `wedding.json` because it is embedded at
compile time via `include_str!`.

## Adding a New Page / Route

1. Create `src/pages/my_page.rs` with a `#[component]` function.
2. Declare it in `src/pages/mod.rs`:
   ```rust
   mod my_page;
   ```
3. Add a variant to the `Route` enum:
   ```rust
   #[route("/my-path")]
   MyPage {},
   ```

## Common Issues

**SCSS not recompiling**
Delete `target/` and rebuild — Cargo sometimes caches the build script result.

```sh
cargo clean && dx serve
```

**`DATABASE_PATH` not found**
Update the `#[cfg(debug_assertions)]` constant in `src/database/mod.rs` to the
absolute path of your local `database.db`.

**`wasm32-unknown-unknown` target missing**
```sh
rustup target add wasm32-unknown-unknown
```

**Port already in use**
```sh
dx serve --port 8081
```
